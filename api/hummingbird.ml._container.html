hummingbird.ml._containerÂ¶

All custom model containers are listed here. In Hummingbird we use two types of containers: - containers for input models (e.g., CommonONNXModelContainer) used to represent input models in a unified way as DAG of containers - containers for output models (e.g., SklearnContainer) used to surface output models as unified API format.
